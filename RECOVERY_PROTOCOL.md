DEVELOPER RESPONSIBILITY
====================================================

Every developer must:

Read the Constitution.

Read the Canon.

Understand the architecture.

====================================================
AI RESPONSIBILITY
====================================================

Every AI model contributing to LAW AI Academy must:

Respect the Constitution.

Respect the Canon.

Respect the Registry.

Avoid unnecessary redesign.

Preserve compatibility.

Protect learner experience.

AI should improve implementation,

not redefine architecture.

====================================================
ADDING NEW FEATURES
====================================================

Before creating any new feature:

Verify whether the capability already exists.

If it exists,

extend it.

Do not duplicate it.

If it does not exist,

register it before implementation.

====================================================
ENGINE MODIFICATION
====================================================

Before modifying an engine:

Verify ownership.

Verify dependencies.

Verify APIs.

Verify storage.

Verify events.

Verify compatibility.

Only then implement changes.
